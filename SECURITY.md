## 报告安全漏洞

我们非常重视安全问题。如果你发现安全漏洞，请**不要**通过公开的 GitHub Issue 报告。

### 报告流程

1. **私密报告**：请通过 GitHub 的私密安全顾问功能报告漏洞
   - 访问：https://github.com/ZestBox-18/CodeFlash/security/advisories
   - 点击 "Report a vulnerability"

2. **提供信息**：
   - 漏洞描述
   - 复现步骤
   - 影响范围
   - 可能的修复方案（如果有）

3. **响应时间**：
   - 我们会在 48 小时内确认收到报告
   - 我们会在 7 天内给出初步评估
   - 我们会在确认漏洞后尽快发布修复

## 安全最佳实践

### 使用建议

1. **网络安全**：
   - 仅在可信的局域网环境中使用
   - 不要将服务暴露在公网
   - 定期检查防火墙设置

2. **权限管理**：
   - 仅授予必要的权限
   - 定期审查访问权限
   - 不要在公共设备上保存凭据

3. **更新维护**：
   - 及时更新到最新版本
   - 关注安全公告
   - 定期检查依赖更新

### 已知安全考虑

1. **局域网传输**：
   - CodeFlash 设计用于局域网环境
   - 数据在局域网中明文传输
   - 不建议在不可信的网络环境中使用

2. **剪贴板访问**：
   - 应用需要访问系统剪贴板权限
   - 请仅从官方渠道下载应用
   - 注意审查剪贴板内容
